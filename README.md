## How I Approach Work

```
Structure First     →  Data model and system design before the first line of code
Clear Code          →  Written for the next developer, not just for today
Async by Default    →  Queues, background jobs, and event handling are not afterthoughts
Fail Gracefully     →  Third-party integrations are failure points — I design around them
No Premature Scale  →  Avoid over-engineering early, avoid shortcuts that become debt
```
